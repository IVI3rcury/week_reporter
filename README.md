# 日志填报系统（Actix Web + rusqlite + SQLite）

这是一个极简的局域网日志填报/周报汇总小工具。

## 功能

### 填报端：`/fill/new`

- 首先选取月份与例会次数
  - 月份：1 月到 12 月。
  - 例会次数：第 1 次例会到第 20 次例会。
- 之后填写想在该次例会中填报的工作项，点击新增工作项可以创建多条工作项，一次可以提交多条工作项。
- 每条工作项包含：
  - 项目号
  - 专业方向
  - 工作模块负责人
  - 上周工作项
  - 工作项填报人
  - 完成进度
  - 原因
  - 本周计划工作项
- 项目号、专业方向、人员只能从选项表中选择，不能自由填写。可由管理员在/summary/options界面配置人员，项目，方向表
- 为应对或许下拉选项框选项过多的情况，填报端的项目号、专业方向、人员选择框带筛选输入框；输入首字母或关键词后，下拉框只保留匹配项。

### 汇总端：`/summary`

- 该页面共3处功能模块：
  - 项目日志汇总模块用于集中展示某次例会中某个项目的整体情况
  - 填报统计模块用于查询某次例会所有人员的填报情况，会将未填报的人员展示出来
  - 日志筛选和日志明细用于以某些组合条件筛选所有历史日志展示

### 项目日志汇总：`/summary/project-report`

- 这是独立功能点。
- 先选择某次例会和某个项目。
- 该汇总页面仍按专业方向分组展示。

### 填报统计：`/summary/attendance`

- 这是独立功能点。
- 先选择某次例会。
- 系统列出现有完整人员名单。
- 按“工作项填报人”判断该人员是否有该次例会的填报记录。
- 表格显示：
  - 人员
  - 是否填报
  - 填报条数
  - 涉及项目
  - 专业方向
  - 最近提交时间

### 选项维护：`/summary/options`

- 维护人员名单。
- 维护项目号。
- 维护专业方向。
- 支持启用、禁用、删除。
- 已被日志引用的选项不允许删除，建议改为禁用；未被引用的选项可以物理删除。

## 技术栈

- Rust
- Actix Web
- rusqlite
- SQLite

## 运行

```bash
cargo run
```
或直接在本项目中的action自动化编译workflow中下载对应架构的绿色版

启动后访问：
访问端口修改为3001
- 填报端：<http://127.0.0.1:3001/fill/new>
- 汇总端：<http://127.0.0.1:3001/summary>
- 项目日志汇总：<http://127.0.0.1:3001/summary/project-report>
- 填报统计：<http://127.0.0.1:3001/summary/attendance>
- 选项维护：<http://127.0.0.1:3001/summary/options>


## 部署

编译后可以部署为：

```text
weekly-log-options-actix-rusqlite.exe
weekly_log.db
```

数据库文件不存在时会自动创建。
