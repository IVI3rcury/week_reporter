# 日志填报系统（Actix Web + rusqlite + SQLite）

这是一个极简的局域网日志填报/周报汇总小工具。

## 功能

### 填报端：`/fill/new`

- 不再使用开始时间/结束时间。
- 改为选择：
  - 月份：1 月到 12 月。
  - 例会次数：第 1 次例会到第 20 次例会。
- 一次可以提交多条工作项。
- 每条工作项包含：
  - 项目号
  - 专业方向
  - 工作模块负责人
  - 上周工作项
  - 工作项填报人
  - 完成进度
  - 原因
  - 本周计划工作项
- 项目号、专业方向、人员只能从选项表中选择，不能自由填写。
- 填报端的项目号、专业方向、人员选择框带筛选输入框；输入首字母或关键词后，下拉框只保留匹配项，仍然不能自由填写。
- 填报端页面不提供跳转到汇总端或选项维护页的按钮。

### 汇总端：`/summary`

- 只显示：
  - 日志筛选框
  - 完整的每条填报日志明细
- 不再在汇总首页显示：
  - 上周工作项汇总
  - 本周计划工作项汇总
  - 记录数、项目数、专业方向数等统计卡片
- 支持按月份、例会次数、项目号、专业方向、人员、完成进度筛选。
- 长文本会自动换行，并在表格单元格内限制高度，避免页面被撑开。

### 项目日志汇总：`/summary/project-report`

- 这是独立功能点。
- 先选择某次例会和某个项目。
- 进入页面后生成：
  - 上周工作项汇总
  - 本周计划工作项汇总
- 该汇总页面仍按专业方向分组展示。

### 填报统计：`/summary/attendance`

- 这是独立功能点。
- 先选择某次例会。
- 系统列出现有完整人员名单。
- 按“工作项填报人”判断该人员是否有该次例会的填报记录。
- 表格显示：
  - 人员
  - 是否填报
  - 填报条数
  - 涉及项目
  - 专业方向
  - 最近提交时间

### 选项维护：`/summary/options`

- 维护人员名单。
- 维护项目号。
- 维护专业方向。
- 支持启用、禁用、删除。
- 配置表不显示内部 ID 和排序字段。
- 新增前会检测重复，已有配置项会直接提示，不执行 INSERT，因此不会占用自增 ID。
- 禁用只影响填报端下拉框，汇总页仍正常显示历史记录。
- 已被日志引用的选项不允许删除，建议改为禁用；未被引用的选项可以物理删除。

## 技术栈

- Rust
- Actix Web
- rusqlite
- SQLite
- 服务端直接返回 HTML，不使用 Vue/React，不需要 Nginx/MySQL/Redis。

## 运行

```bash
cargo run
```

启动后访问：

- 填报端：<http://127.0.0.1:3000/fill/new>
- 汇总端：<http://127.0.0.1:3000/summary>
- 项目日志汇总：<http://127.0.0.1:3000/summary/project-report>
- 填报统计：<http://127.0.0.1:3000/summary/attendance>
- 选项维护：<http://127.0.0.1:3000/summary/options>

## 插入示例数据

PowerShell：

```powershell
$env:SEED_SAMPLE="1"; cargo run
```

Linux/macOS：

```bash
SEED_SAMPLE=1 cargo run
```

## 部署

编译后可以部署为：

```text
weekly-log-options-actix-rusqlite.exe
weekly_log.db
```

数据库文件不存在时会自动创建。

## 旧版数据库兼容说明

新版使用 `meeting_month` 和 `meeting_count` 表示“XX 月第 X 次例会”。
程序启动时会尝试给旧版 `weekly_logs` 表补充这两个字段；旧记录会默认归到 `1 月第 1 次例会`，后续可以在编辑页面手动调整。
